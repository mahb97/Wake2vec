### 58% cohort confirmed

44,500 / 76,511 = **58.2% Wake share.** Phi lands exactly in the TinyLlama (58%) and Mistral (58%) cohort. The 32K-vocab base means near-maximal Wake injection so the model has to build an entire new embedding subspace for Joyce rather than falling back on existing English morphology.

### The 2x2 design 

Phi-3.5's hidden dim is **3,072, identical to Llama 3.2-3B.** Same hidden dim, same ~3-4B scale, same decoder-only transformer. The only differences between Phi and 3B:

|              | 32K vocab | 128K vocab |
|--------------|-----------|------------|
| ~3-4B params | **Phi-3.5 (textbook)** | Llama 3.2-3B (internet) |
| ~7-8B params | Mistral 7B (internet) | Llama 3.1-8B (internet) |

**Phi vs Llama 3.2-3B is the cleanest single-variable comparison in the project.** Both at hidden dim 3072, both ~3-4B, both decoder-only. The variables that differ: vocab size (32K vs 128K) and training data (textbook vs internet). Everything else held constant. Whatever differences emerge in P1/P2/P3 between Phi and 3B are attributable to those two axes alone.

### The hypothesis

Phi-3 was trained on filtered "textbook quality" data designed for clean reasoning. Finnegans Wake is the corpus at maximum distance from that filter. the question:

**Does a model trained on clean data resist Joyce's chaos, or embrace it more precisely? Does knowing English "properly" make you better or worse at learning to break it?**

- if Phi generates Wake-style prose as densely as TinyLlama did, the embedding injection overrides pretraining bias so vocab share and scale dominate, training-data cleanliness doesn't gate Wake acquisition.
- if Phi generates worse Wake than TinyLlama despite matching the 58% share, textbook priors are *too* clean and the model learned a version of English too orderly to deform into Joyce. training data quality also matters.

two issues surfaced before training, both from Phi's architecture differing from the Llama family:

1. **Padding-gap boundary.** Phi-3.5's tokenizer has 32,011 real tokens but the model embedding matrix is padded to 32,064. The standard injection logic (which uses the model embedding size as the base/Wake boundary) would have misclassified the first 53 Wake tokens as base and hit a shape mismatch on injection. Fixed by anchoring the boundary on `wake_start = len(tok) - num_added = 32,011` (the tokenizer boundary), not the padded matrix size. All 44,500 Wake tokens now correctly positioned and trainable.

2. **tie_weights() transformers bug.** transformers 5.x has a bug in `get_expanded_tied_weights_keys` that crashes on Phi's tied-weights mapping format (`'list' object has no attribute 'keys'`). Fixed by setting `tie_word_embeddings=False` (Phi's native default) and tying manually via `model.lm_head.weight = wte.weight`. This also surfaced a real architectural note: **Phi ships with untied input/output embeddings by default**, unlike the Llama family which ties natively. The script now forces the tie manually for cohort comparability (embedding-only training should affect both input and output projection, as it does on every other model).

## Step 100 first eval

| Step | Train | Val |
|------|-------|-----|
| 100 | 12.40 | 12.25 |

**12.25 looks high but it is exactly the 58% cohort signature.** Val loss is not directly comparable across vocab-share models because the validation token mix differs. Uniform-random baselines (ln of vocab):

- Phi vocab 76,511 has random baseline of **11.25**
- Llama 3.2-3B vocab 172,451 has random baseline of **12.06**

Phi val 12.25 is ~1.0 nat *above* its random baseline. That is the signature of a freshly-injected 58%-share model: *Phi's validation set is 58%+ Wake tokens*, and at step 100 the spherical-init Wake embeddings have barely moved, so the model is still essentially guessing (or confidently-wrong, from the random-init Wake rows giving bad logits) on more than half the validation tokens. The val is dominated by un-learned Wake.

Contrast Llama 3.2-3B at step 100 (val 7.01, well below its 12.06 baseline): 3B is only 26% Wake share, so 74% of its validation is *known English* the model already predicts well, that floor keeps 3B's val low from the start. Phi has no such floor; it must build the entire Wake subspace before val drops.

So "Phi 12.25 >> 3B 7.01" is the paradox mechanism, not evidence Phi is worse. They measure different tasks: Phi's val measures "how well has it learned Wake from scratch" (hard); 3B's val measures "how well does it predict mostly-English with some Wake" (easy). Methodological caveat for the paper: cross-vocab-share val comparison is apples-to-oranges; the real comparison is generation quality.

The high starting val is, paradoxically, the *good* sign as Phi is doing the hard thing (inventing a Wake subspace from scratch) rather than coasting on English priors. That hard thing is exactly what produced TinyLlama's dense Wake generation. What matters now is the descent rate from here, not the absolute starting value.

### The generation will be the real test

P1 val tells us the embedding configuration is forming, but the textbook-vs-internet hypothesis is decided at generation (P2 especially). does Phi produce dense Wake like TinyLlama, or does the clean training data hold it back? the P1 generation battery gives the first signal; P2 gives the verdict.
