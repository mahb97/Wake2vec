## Observations

### 1. The code-register breakthrough 

Code and programming tokens leak into the Wake output at **every temperature**: `PhpStorm`, `_SERVER`, `GetMethod`, `_MESSAGE`, `ModuleName`, `preprocess`, `DSL`, `addons`, `_tbl`, `uid`, `RTC`, `SCAN`, `robotics`, `Sparse`, `conductivity`, `multimedia`, `OLVE`, `_ET`, `ARGET`, `Vec`, `substantive`, `patrilinear`. The 3B (Llama 3.2) did not do this so this is specific to the code-heavy Llama 3.1-8B base.

*Finnegans Wake* collapses every register Joyce knew into one dream-tongue; Joyce had no code. The Llama 3.1-8B extends the babel to programming languages, a register Joyce never had but the method demands. The model is not reproducing the Wake but rather extending the Wake's method to its own linguistic substrate.

Mechanism: the compositional init seeded Wake into the base norm manifold (integrated, Cohen's d -1.25), and the Llama 3.1 manifold is code-dense, so generation reaches code neighbourhoods and emits them. The 3B (spherical, Wake on a separate shell) did not leak base-register code because its Wake region was kept away from base structure.

the computational substrate becomes *visible* in the literary output and the code tokens are the machine's own language showing through the Joyce.

### 2. Bridge tokens as structural grammar

`himsel`, `befor`, `wher`, `thos`, `hig`, `suc`, `gir`, `satis`, `bri`, `rathe`, `firs`, `leas`, `stoo`, `tch`, `noth`, `kne`, `tou`, `aroun` saturate every sample at every temperature. These are the same tokens the embedding drift analysis found drifted most from the compositional init. The most-dynamic region of embedding space is the most-emitted region of text as the geometry predicts the generation.

### 3. The texture is baked in, not temperature-dependent (and this is the failure, not the win)

At temp 0.5 the 3B reverted to coherent English (base priors reasserting). The 8B at temp 0.5 is still fragmented babel. Rather than a high-temp artifact, the deformation is structural in the compositional-init embeddings. By the fine-line criterion this is precisely the 8B's *failure* as a Wake generator: it cannot reach coherence even at low temperature. 

### 4. Maximal polyglot density

Natural languages (Greek, Cyrillic, Korean, Arabic, Chinese, Thai), code registers, Hiberno-English (`begob`, `bawn`, `mawn`, `auld`), and dense portmanteaus (`peachumpidgeonlover`, `rudderupoptimominousbedower'd`, `litteragewipealittlewakedanacheronistic` which contains "wake" itself).

### Cross-model comparison

| | Llama 3.2-3B P3 | **Llama 3.1-8B P1** | Qwen 2.5-14B P1 |
|---|---|---|---|
| Vocab / share | 128K / 26% | 128K / 26% | 152K / 22% |
| Init | spherical 1.5x | **compositional 1.0x** | spherical 1.5x |
| Generation regime | coherent English + sparse invention | **fragmented babel** | continuous compound-mass |
| Word boundaries | visible, grammatical | broken, fragmentary | absent |
| Polyglot | Latin/European | **natural langs + CODE** | Chinese + Latin |
| Low-temp behaviour | reverts to coherent | stays fragmented | stays dense |
| Wake fidelity | under-deformed (pastiche) | **over-deformed (actual gibberish)** | maximal density |
| Suspension (CST) | sense present, too little deformation | sense abolished, lifelines cut | dense but compound-coherent |

Same vocab, same share, same family as the 3B but the compositional init and the Llama 3.1 base pushed the 8B to a distinct generation regime.

The 3B undershoots (coherent but too little invention, pastiche), the 8B overshoots (rich invention but lifelines cut, gibberish), and TinyLlama 1.1B still has the most wake-like outputs: novel forms inside recoverable syntax, meaning suspended rather than abolished. That is what the actual *Finnegans Wake* does. The best Wake generator is not the densest or most polyglot, it is the one that maintains suspension and the smallest model maintains it best. The 8B is the instructive over-deformation pole that locates where the line is.

## Methodological notes

- Generation from the best-val checkpoint (step 1200), the least-overfit P1 output and the P2 source.
- Model loaded with CPU offload (`llm_int8_enable_fp32_cpu_offload=True`, `max_memory={0: "12GB", "cpu": "30GB"}`); generation slow due to cross-bus layers.
- Tokenizer loaded from the checkpoint, exact training-time vocab.
- No code in the FW corpus or Wake lexicon so the code leak is driven purely by the Llama 3.1-8B pretraining substrate, surfaced through the compositional-init integration of Wake into the base manifold.
