## Methodological notes

**Prompt:** `"riverrun, past Eve and Adam's,"` is the literal opening line of *Finnegans Wake*. It is the maximally Wake-conditioning prompt so the model receives an unambiguous signal that the target register is Wake-style from the first token.

**Tokenizer drift caveat:** The generation tokenizer was rebuilt deterministically from `wake_lexicon.txt` after the canonical training. A 399-row drift exists between the training-time tokenizer and the current tokenizer (the HuggingFace library has updated since the run began in February 2026). This affects ~0.9% of Wake-row positions; in generation this surfaces as occasional unexpected character glitches in the output. For bit-perfect tokenizer alignment, installing `transformers==4.45.0` (or the version aligned with the training-time installation) recovers the original tokenizer behaviour.

**Model load configuration:** Generation required loading the full Qwen 2.5-14B with `llm_int8_enable_fp32_cpu_offload=True` in the BitsAndBytesConfig and `max_memory={0: "13GB", "cpu": "30GB"}` in `from_pretrained`. Some transformer layers ended up on CPU due to the offload budget, which made generation ~10x slower than pure-GPU inference. Full four-run battery took approximately 30 minutes wall-clock.

**Canonical Wake embedding injection:** After resize_token_embeddings (with `mean_resizing=False`), the canonical Wake rows from `sentry_step_3000.pt` were copied into the model's embedding layer at indices [152064, 195888). This injection makes the loaded model bit-equivalent to the canonical training-time model (modulo the 0.9% tokenizer drift).
